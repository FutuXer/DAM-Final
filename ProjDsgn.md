项目启动规划书：基于多源时序特征融合的信用风险预测 (Home Credit)

一、 项目定位与设计思路

本项目完全摒弃复杂的系统开发，将全部算力与精力倾注于特征工程（Feature Engineering）与算法调优。

核心痛点：传统的信用评分仅依赖用户填写的静态申请表，难以真实反映其动态还款能力。

破局思路（多源/多维融合）：将申请人的“静态切片数据”（主表）与外部征信局的“历史跨度数据”（Bureau）、内部的“高频交易/还款流水数据”（Installments/POS）进行多源深度融合。

算法极化对比：针对高度不平衡的违约数据（违约率通常不到 10%），系统性评估至少 5 种机器学习算法在真实金融风控场景下的性能与解释性。

二、 核心数据集拆解与特征融合战略 (Data Pipeline)

本项目数据集包含多张关联表，呈现非平铺结构。我们将通过特征工程将其整合为单张宽表以供模型训练。

1. 主表：静态画像基座 (Base)

application_train.csv：主干数据（SK_ID_CURR 为主键）。

处理动作：处理缺失值、异常值（如 DAYS_EMPLOYED 中异常的 365243 天），对分类变量（职业、学历、房产）进行 One-Hot 编码或 Target Encoding。

2. 外部征信源：低频动态特征 (External Bureau)

bureau.csv & bureau_balance.csv

处理动作：不能直接 Join。必须针对每个 SK_ID_CURR，利用 groupby 进行统计聚合（Aggregations）。

衍生特征示例：提取该用户在外部征信局的“历史贷款总次数”、“历史逾期平均天数”、“活跃状态贷款的占比”。

3. 内部高频流水源：高频时序特征 (Internal Behaviors)

POS_CASH_balance.csv、credit_card_balance.csv、installments_payments.csv

处理动作：这是拿高分的关键。挖掘用户近期的行为轨迹。

衍生特征示例：提取“最近6个月内信用卡平均透支额度”、“历史分期付款中实际还款金额与应还金额的差值（欠款倾向）”、“提前还款天数的均值”。

三、 期望的项目结果 (交付物)

超高维特征宽表：从原始的几十个特征，通过交叉聚合，繁衍出包含几百上千个有效特征的训练矩阵（极其体现数据挖掘功底）。

5种算法对比评估表：在统一的数据集和交叉验证（CV）策略下，输出 5 种算法的对比矩阵。

风控模型解释图谱 (SHAP)：金融风控必须可解释。使用 SHAP Value 图表，向评委清晰展示：到底是什么特征（比如外部评分、近期逾期天数、年龄）最容易导致用户违约。

四、 核心功能点与 Pipeline

1. 数据清洗与特征繁衍流

缺失值填充与离群点盖帽。

特征聚合 (Aggregation)：算子包括 mean, max, min, sum, count, var。

特征选择 (Feature Selection)：由于衍生特征可能高达上千个，必须使用基于树模型的特征重要性、相关性分析（Pearson）或 L1 正则化进行特征降维。

2. 算法建模流 (5种算法对比)

本项目是二分类问题，正负样本极度不平衡（TARGET=1 极少）。

基线模型 (Baseline)：Logistic Regression (逻辑回归) - 弱能力，强解释。

树模型基础：Random Forest (随机森林) - 强鲁棒性。

大杀器1：XGBoost - 传统的金融风控王者。

大杀器2：LightGBM - 处理海量数据与类别特征速度极快，Kaggle 标配。

大杀器3：CatBoost (可选) - 专门针对 Categorical 类别特征优化的神器；或选择简单 MLP 神经网络作为对比。

3. 评估指标设计 (Evaluation)

禁用：Accuracy（准确率）。因为全猜 0 也能有 92% 准确率。

必须使用：AUC-ROC 曲线（衡量排序能力）、Recall（召回率，抓违约客户的能力）、F1-Score。

五、 团队角色建议 (基于4人编制 - 精准分工)

本项目的分工极其好切分，大家可以分别负责不同的“数据表”。

队员 A (主表特征工程师)：负责清洗 application 主表，处理所有的异常值与类别编码；负责最后的特征合并。

队员 B (外部时序特征工程师)：专门负责“啃” bureau 和 bureau_balance 这两张表，写 groupby 提取外部信用特征。

队员 C (内部流水特征工程师)：专门负责提取 installments（分期还款）和 POS 等高频行为流水特征（这里能挖出最强特征）。

队员 D (算法与评估总监)：等前三个人把特征宽表交给你后，负责写 5 种模型的训练代码，调参（GridSearch/Optuna），画 ROC 曲线和 SHAP 解释图，统稿最终报告。
