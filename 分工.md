## 二、队员 A —— 数据工程

### 要写的代码（5个文件）

| 文件 | 路径 | 做什么 |
|------|------|------|
| `download_poi.py` | `src/data/` | 从 HOTOSM 下载日本 POI GeoJSON，筛选 tourism/restaurant/shop/hotel 四类，输出 CSV |
| `download_census.py` | `src/data/` | 从 e-Stat SSDSE 下载 1741 市区町村统计数据，提取人口/密度/老龄化/服务业从业人数 |
| `preprocess.py` | `src/data/` | 经纬度 Min-Max 归一化、POI 类别 One-Hot 编码、缺失值处理 |
| `entity_alignment.py` | `src/data/` | L0 空间匹配（评论坐标↔POI <50m）+ L1 名称精确匹配 + L2 模糊匹配 |
| `feature_merge.py` | `src/data/` | 将 Geo + Text + Transit + Demo 四源特征按 POI ID 拼接，输出特征宽表 |

### 要产出的数据文件

| 文件 | 路径 | 说明 |
|------|------|------|
| `pois_clean.csv` | `data/processed/` | 筛选 + 归一化后的 POI 表 |
| `reviews_aligned.csv` | `data/processed/` | 实体对齐后的评论-POI 映射表 |
| `census_clean.csv` | `data/processed/` | 清洗后的区域统计表 |
| `feature_matrix.csv` | `data/processed/` | ★ 最终四源特征宽表，这是全组的核心输入 |

### 要写的文档

| 文件 | 路径 | 说明 |
|------|------|------|
| `data_dictionary.md` | `docs/` | 特征宽表每个字段的中文名称、取值范围、数据来源 |
| 报告"数据与预处理"章节 | `report/report.pdf` | 数据来源说明、清洗流程、实体对齐覆盖率、宽表统计概览 |

## 三、队员 B —— 算法建模

### 要写的代码（5个文件）

| 文件 | 路径 | 做什么 |
|------|------|------|
| `download_reviews.py` | `src/models/` | 用 `datasets` 库加载 HuggingFace `itinerai/attractions` + `ACOSRes`，输出清洗后 CSV |
| `nlp_vectorizers.py` | `src/models/` | 实现 TF-IDF（Top-500 keys）、Word2Vec（200d）、MiniLM（384d）三种向量化 |
| `distance_metrics.py` | `src/models/` | 实现联合距离公式：`D_joint = α·D_geo_norm + (1-α)·(1-Sim_text)` |
| `clustering.py` | `src/models/` | K-Means / DBSCAN / Agglomerative，均支持自定义 distance metric |
| `grid_search.py` | `src/models/` | α∈{0.1,0.3,0.5,0.7,0.9} × K∈[3,15] × eps 网格搜索 |

### 要产出的数据文件

| 文件 | 路径 | 说明 |
|------|------|------|
| `reviews_raw/` | `data/raw/` | HuggingFace 原始下载缓存 |
| `grid_search_results.csv` | `outputs/results/` | 所有参数组合的 Silhouette/DBI 汇总 |
| `best_model_params.json` | `outputs/results/` | 最优 α、K 值、NLP 方案 |

### 要写的文档

| 文件 | 路径 | 说明 |
|------|------|------|
| 报告"方法"章节 | `report/report.pdf` | NLP 方案选型分析、距离公式推导、聚类算法原理与参数、寻优策略 |

## 六、W1 周一行动清单（每人一小时）

```
A: 下载 HOTOSM GeoJSON → CSV → 筛选 POI 类别 → 群里发数据快照

```

**W1 周三晚同步会（30min）**：每人汇报数据获取结果，D 确认所有人已 push 第一版代码。

## 七、协作规则

- **分支**：`main`（保护） + `dev-A` / `dev-B` / `dev-C` / `dev-D`，合并前至少一人 review
- **提交**：每人每天至少一次 commit（含 WIP 也行），避免最后一周突击
- **接口**：A 与 B 对接 `feature_matrix.csv` 的列名和数据格式；B 与 C 对接聚类结果输出的数据结构
- **文档**：每个 `.py` 文件头写一行功能说明；关键函数写 docstring
- **沟通**：遇到卡点立刻在群里说，不要自己闷头搞两天
