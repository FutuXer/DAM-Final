## 6. 常见问题

### Q1: 运行 Notebook 时提示 "No module named 'catboost'"

**原因**: 内核选择的 Python 环境中没有安装 catboost。

**解决**: 在 VS Code 右上角点击"选择内核"，选择系统 Python 3.10（`C:\Users\Administrator\AppData\Local\Programs\Python\Python310\python.exe`），而不是 conda 的 base 环境。

### Q2: Cell 运行很久没有输出

这是正常的，因为：
- SMOTE Cell：24 万样本 × 387 特征，需 3-5 分钟
- 5-Fold CV Cell：每个模型训练 5 次，需 15-20 分钟
- XGBoost GridSearch Cell：搜索 3×3×3×3=81 种参数组合，需 3-5 分钟

可以通过 Cell 右上角的旋转图标判断是否仍在运行。

### Q3: 内存不足怎么办？

如果遇到内存错误（`MemoryError`），可以：
1. 减小 SMOTE 样本量：在 Cell 6 中设置 `SMOTE(sampling_strategy=0.5)` 只生成 50% 的正样本
2. 减少 CV 折数：在 Cell 20 中将 `n_folds=5` 改为 `n_folds=3`

### Q4: 如何重新生成 `final_feature_matrix.csv`？

如果修改了预处理代码需要重新生成最终宽表，在 Notebook 的 Cell 4 之前手动运行：

```python
from src.data.preprocess_application import merge_with_features
merge_with_features(
    'data/processed/processed_application_train.csv',
    [
        ('data/processed/features_bureau.csv', '队员B'),
        ('data/processed/features_internal.csv', '队员C'),
    ]
)
```

然后再继续运行 Notebook。
